## ЁЯТб Things I Learned the Hard Way

> тЪая╕П These are hard lessons тАФ save yourself the pain!

- Always test your SSH keys **before** adding them to Jenkins credentials
- Docker containers are **not** VMs тАФ they share the host kernel
- `git rebase` rewrites history тАФ never rebase a shared/public branch
- Port conflicts in Docker are easy to miss тАФ always check with `docker ps`
- Cron job timing syntax has **5 fields** тАФ count them every time
- A misconfigured security group in AWS will silently block all traffic
- `EXPOSE` in Dockerfile is documentation only тАФ use `-p` to actually publish
- Always run a `trivy image` scan **before** pushing to Docker Hub to avoid security vulnerabilities
- If the SonarQube Quality Gate fails, stop the pipeline immediately тАФ never deploy unverified code
- The OWASP Top 10 is not just theoretical; most real-world attacks exploit these exact vulnerabilities
- Jenkins doesnтАЩt behave the same across pipeline types тАФ things like `BRANCH_NAME`, default checkout, and execution context can silently break logic if not handled properly.
- Running Jenkins inside Docker doesnтАЩt mean Docker is available inside it тАФ you need explicit access (like mounting the Docker socket), or your pipeline will fail at runtime.
